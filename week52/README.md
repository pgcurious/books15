## The Three Frameworks United

Throughout this course, we've applied three thinking frameworks. In this final week, we see how they are not separate lenses but aspects of a unified understanding.

### First Principles: The Atoms of Agency

**The Question**: What is irreducibly necessary for agency to exist?

After 52 weeks, we can answer with precision:

```
AGENCY = PERCEPTION + REASONING + ACTION + LEARNING

At the atomic level:
├── PERCEPTION: Transforming raw input into structured understanding
│   └── Implemented via: Embeddings, context windows, retrieval
│
├── REASONING: Generating novel inferences from understanding
│   └── Implemented via: LLMs, chain-of-thought, planning algorithms
│
├── ACTION: Affecting state changes in the environment
│   └── Implemented via: Tools, APIs, code execution, physical actuators
│
└── LEARNING: Updating behavior based on outcomes
    └── Implemented via: Memory systems, fine-tuning, RLHF, self-reflection

Remove any one, and agency collapses:
├── No perception → Blind action (automation, not agency)
├── No reasoning → Fixed responses (scripts, not agents)
├── No action → Passive observation (models, not agents)
└── No learning → Static behavior (software, not intelligence)
```

**The Insight**: Every agent architecture, from the simplest ReAct loop to the most complex multi-agent system, is a variation on these four primitives. Mastery comes from understanding how to compose them for your specific context.

### Emergence Thinking: Complexity from Simplicity

**The Question**: How does sophisticated behavior arise from simple rules?

The central mystery of intelligence—biological or artificial—is emergence. Our year of study reveals the pattern:

```
Layer 0: Tokens
└── Simple symbols, no meaning in isolation

Layer 1: Embeddings
└── Tokens gain meaning through relationship to other tokens
    EMERGENCE: Semantic understanding

Layer 2: Attention
└── Embeddings relate dynamically based on context
    EMERGENCE: Contextual reasoning

Layer 3: Prompting
└── Attention is directed toward specific goals
    EMERGENCE: Task performance

Layer 4: Tool Use
└── Reasoning connects to external capabilities
    EMERGENCE: World interaction

Layer 5: Memory
└── Interactions persist and accumulate
    EMERGENCE: Continuity and learning

Layer 6: Multi-Agent
└── Multiple agents interact and coordinate
    EMERGENCE: Collective intelligence

Layer 7: Self-Improvement
└── Agents modify their own behavior
    EMERGENCE: Autonomous growth

Layer 8: ???
└── We are here, building the next layer
    EMERGENCE: Unknown, but we are its architects
```

**The Insight**: Each layer emerges from the interaction of elements in the layer below. Intelligence is not designed at the top level—it crystallizes from the bottom up. Your role as an agent architect is to create the conditions from which intelligence emerges.

## The Capstone Philosophy

This week, you'll build a final capstone that embodies everything you've learned. But first, understand what makes a capstone meaningful:

### A Capstone Is Not Just a Project

```
PROJECT: "I built something that works"
         └── Demonstrates technical skill

CAPSTONE: "I built something that reveals my understanding"
         └── Demonstrates integrated mastery
         └── Shows design philosophy
         └── Reflects on trade-offs
         └── Anticipates failure modes
         └── Considers ethical implications
         └── Points toward future possibilities
```

### Capstone Design Principles

```
1. INTEGRATION
   └── Combine multiple techniques in meaningful ways
   └── Show how components amplify each other

2. DEPTH OVER BREADTH
   └── Better to do one thing excellently
   └── Than many things superficially

3. CLEAR PHILOSOPHY
   └── Every design choice should be justified
   └── "Why this way?" matters as much as "what?"

4. HONEST LIMITATIONS
   └── Know what your system cannot do
   └── Identify failure modes explicitly

5. FUTURE ORIENTATION
   └── How would this evolve?
   └── What would make it 10x better?

6. ETHICAL GROUNDING
   └── Who does this help?
   └── Who might this harm?
   └── What safeguards exist?
```
